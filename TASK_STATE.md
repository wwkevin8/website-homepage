# TASK STATE

## 1. 当前项目目标

当前项目是 `NGN` 的前台服务站点 + 后台管理系统，重点围绕两条业务线持续迭代：

- 接送机服务
  - 学生前台查看接送机服务介绍
  - 提交接机 / 送机 / 拼车需求
  - 查看公开拼车信息并判断是否适合加入
  - 后台登记、编辑、分组、改价、付款、邮件通知
- 寄存服务
  - 学生前台查看寄存服务页
  - 进行寄存预约
  - 后台继续保留寄存管理能力

当前工作的核心目标是：

- 保持前台公开页易懂、简洁、适合学生快速判断
- 保持后台管理页适合运营快速处理订单和拼车组
- 在不升级 Vercel Pro 的前提下，继续稳定上线和维护生产环境

## 4. 关键约束和不能改动的部分

### 业务约束

- 前台个人中心里的账号邮箱不能修改
  - 原因：注册与邮箱验证基于账号邮箱
- 后台订单页里的邮箱是“订单联系邮箱”
  - 可以编辑
  - 用于付款确认邮件等运营联系
- 一张订单只能属于一个拼车组
- 拼车组成员不能直接移出后变成“无组状态”
  - 应走“更换到其他拼车组”的流程
- 前台公开页不能暴露隐私字段
  - 不展示姓名、电话、微信、详细地址、后台状态等

### 上线约束

- 当前部署仍使用 Vercel Hobby
- 必须控制 Serverless Functions 数量，不能随意把公开 API 再拆回去
- 公开前台接口目前通过：
  - `E:\webside\api\public\[...action].js`
  - `E:\webside\public-api-handlers\*.js`
  来做统一分发
- `vercel.json` 中 cron 当前必须保持轻量，避免再次触发套餐限制

### 前端结构约束

- 前台拼车详情弹窗要保持现有整体 UI 风格
- 不要为了新需求大改整页视觉，只调整字段、结构、排版
- 后台页面虽然可持续优化布局，但不能破坏现有录单和运营习惯

### 数据与数据库约束

- `transport_requests` 表现在已新增 `email` 列
- 后台订单联系邮箱保存依赖这个列，不能再删掉
